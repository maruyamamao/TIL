## 基本制御構造
プログラムの構造の基本となるのは
- **順次**
- **選択（分岐）**
- **繰り返し（ループ）**
の3つの制御構造で、これらを**基本制御構造**と言う
基本制御構造だけを用いて読みやすいプログラムを書く考え方を**構造化プログラム**と言う

## 分岐（選択）
条件によって実行する処理を分けることを**分岐(選択)**と呼ぶ
if文を用いる

#### if文のフローチャート
```mermaid
graph TD
  start[開始] --> checkCondition{条件判定}
  checkCondition -- Yes --> processValid[処理1]
  checkCondition -- No --> processInvalid[処理2]
  processValid --> endNode[終了]
  processInvalid --> endNode[終了]
```
#### if文の擬似言語
```html
if(条件式)
　処理1  <!--条件が成立した時の処理-->
else
  処理2  <!--条件が成立しなかった時の処理-->
endif
```

## 複合条件、二分岐

複数の条件式を**and**や**or**で結んだもの

- andは複数の条件がすべて満たされた時だけYESへ進む、１つでも満たさなければNOへ進む
```mermaid
graph TD
  start[開始] --> checkCondition{Age >= 20 and Age <= 29}
  checkCondition -- Yes --> processValid[Div ← 20代です]
  checkCondition -- No --> processInvalid[Div ← 20代以外です]
  processValid --> endNode[終了]
  processInvalid --> endNode[終了]
```
```html:andの場合
Div ← "20代以外です"
if((Age が 20 以上) and (Age が 29 以下))
  Div ← "20代です"
endif
```

- orはどれかの条件が満たされたらYESへ進む、すべて満たされていなかった場合のみNOへ進む
```mermaid
graph TD
  start[開始] --> checkCondition{アイス or パンケーキ}
  checkCondition -- Yes --> processValid[sw ← 甘いスイーツが好き]
  checkCondition -- No --> processInvalid[sw ← 甘いスイーツが好きではない]
  processValid --> endNode[終了]
  processInvalid --> endNode[終了]
```
```html:orの場合
if((LikeFood が アイス) or (LikeFood が パンケーキ))
  sw ← "甘いスイーツが好き"
else
  sw ← "甘いスイーツが好きではない"
endif
```

## 多分岐
3つ以上に分岐先が分かれる構造を**多分岐**と呼ぶ
```mermaid
graph TD
  start[開始] --> checkCondition{Int == 0}
  checkCondition -- Yes --> processValid1[X ← 100]
  checkCondition -- No --> processInvalid{Int= 1}
  processInvalid{Int == 1} -- Yes --> processValid2[Y ← 500]
  processInvalid{Int == 1} -- No --> processValid3[z ← 800]
  processValid1 --> endNode[終了]
  processValid2 --> endNode[終了]
  processValid3 --> endNode[終了]
```
```html:多分岐
if(Int == 0)
　X ← 100
elseif(Int == 1)
　Y ← 500
else
　z ← 800
endifdf
```
